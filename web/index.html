What you are looking at

  1. Pick an alerting service on the left (default faultline-victim-frontend). The scenario hint tells the agent what symptom class to expect when telemetry quota throttles.
  2. Click Start investigation. The Faultline agent (Gemini 2.5 Flash on Vertex AI, orchestrated by Google ADK) runs an 8-step policy: reads Cloud Monitoring metrics, walks the service dependency graph, queries the GitLab MCP server for recent commits, reads the suspect diff, and explains the causal chain.
  3. Cards stream in live as the agent reasons. This takes ~30-90s for a real run because each Vertex AI call + MCP roundtrip is a real network hop.
  4. When the agent identifies the offending commit, it opens a postmortem issue and stages a DRAFT rollback MR in GitLab. The rollback card appears at the bottom-left with an Approve button.
  5. Click Approve. The server strips the Draft: prefix and merges the MR via GitLab REST. The victim recovers.

The agent never merges on its own. A human always clicks Approve. See the policy →
